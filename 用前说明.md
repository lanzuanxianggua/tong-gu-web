## 常用命令

### 服务管理
```bash
# 查看服务状态
sudo systemctl status nginx
sudo systemctl status tong-gu
sudo systemctl status mysql

# 重启服务
sudo systemctl restart nginx
sudo systemctl restart tong-gu
sudo systemctl restart mysql

# 查看日志
sudo journalctl -u tong-gu -f
sudo tail -f /var/log/nginx/tong-gu-error.log
```

### 数据库
```bash
# 备份数据库
mysqldump -u root -p tong-gu > backup_$(date +%Y%m%d).sql

# 恢复数据库
mysql -u root -p tong-gu < backup_20260326.sql

# 查看数据库大小
mysql -u root -p -e "SELECT table_schema, SUM(data_length + index_length) / 1024 / 1024 AS 'Size (MB)' FROM information_schema.tables GROUP BY table_schema;"
```

### 日志清理
```bash
# 清理旧日志（保留最近 7 天）
find /var/log -name "*.log" -mtime +7 -delete
find /var/www/tong-gu-web/logs -name "*.log" -mtime +7 -delete
```

## 故障排查

### 网站无法访问
1. 检查 Nginx 状态：`sudo systemctl status nginx`
2. 检查后端服务：`sudo systemctl status tong-gu`
3. 检查防火墙：`sudo ufw status` 或 `sudo firewall-cmd --list-all`
4. 检查端口监听：`sudo netstat -tulpn | grep :80`

### API 返回 502 Bad Gateway
1. 检查后端服务是否运行
2. 检查 Gunicorn 绑定地址和端口
3. 查看后端日志：`sudo journalctl -u tong-gu -f`
4. 检查 Nginx 配置中的 proxy_pass 地址

### 静态文件 404
1. 运行 `python manage.py collectstatic --noinput`
2. 检查 Nginx 配置中的 static 路径
3. 检查文件权限：`ls -la /var/www/tong-gu-web/static/`

### 图片上传失败
1. 检查媒体文件目录权限
2. 检查 Nginx 的 client_max_body_size 配置
3. 检查 Django 的 DATA_UPLOAD_MAX_MEMORY_SIZE 配置
4. 查看错误日志
